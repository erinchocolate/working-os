# skill: 润色 —— 改英语 + 学主人的英语风格（循环 A）

**触发**：
- 「润色」+ 粘一段英文 → 给润色版。
- 「润色 收」+ 粘主人改过的版本 → 学习主人的风格。

**目的**：改语法、让表达更地道；并通过对比，越来越懂主人本人的英语表达方式，让产出更像主人会说的话。

## 先读
`style/english-profile.md` —— 主人已沉淀的英语风格规则（规则化、可直接当 prompt 用）。

## 路径约束
- 所有润色相关的长期文件只允许写在 `style/` 下。
- 风格规则只更新 `style/english-profile.md`。
- 配对样本只写到 `style/english-samples/`。
- 不要在 `skills/` 下创建任何 `style/` 子目录或风格文件。

## 流程 A：润色（给 v0）
1. 读 `style/english-profile.md`，把已知风格规则套进去。
2. 产出：
   - **改正**：语法/拼写错误。
   - **建议**：3–5 条让表达更地道的具体建议。
   - **润色版**：一个完整的润色后版本，已尽量贴合主人风格。
3. 提示主人：改完后可以用「润色 收」把你的最终版喂回来，我来学。

## 流程 B：润色 收（学风格）
1. 拿到主人的最终版（v1）。
2. **对比 [我的润色版 v0] vs [主人的 v1]**，提炼语义层面的差异：用词偏好、句长、直接 vs 委婉、语气、结构、惯用表达。
3. **把差异提炼成规则**（不是堆原始 diff！），追加进 `style/english-profile.md`。规则要能直接当 prompt 用，例：「偏好短句」「避免被动语态」「不用 'utilize'，用 'use'」。
4. 配对存档：`style/english-samples/<日期>-<简述>.md`，内含 v0 与 v1 两版，供日后回看。

## 原则
`style/english-profile.md` 是**规则化的风格指南**，越用越准；它会在「流程 A」里被注入。不要把它写成 diff 堆砌。
