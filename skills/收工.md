# skill: 收工 —— 把一次 AI 对话或任务 session 总结进 worklog

**触发**：主人说「收工」「结束总结」「总结这次 session」「session digest」。

**目的**：把一次 Codex/Agent 对话里真正值得带走的内容沉淀成 `worklog/`，供日复盘和系统复盘读取。不要把 AI 对话总结放回 `inbox/`。

## 先读
1. `AGENTS.md`（确认目录边界）。
2. 若 session 明显属于某项目，读 `projects/<项目>/context.md` 和 `projects/<项目>/project.md`。
3. 若 session 是系统设计/摩擦讨论，读相关 `skills/*.md`。

## 流程
1. 回顾当前对话或任务，提取：主题、重要想法、形成的决策、待办、可能路由进项目 context 的内容、可能进入复盘的认知。
2. 写 `worklog/<日期>-<简述>.md`。
3. 如果有项目事实、决策、坑已经足够明确，可按「摄取/执行」规则回写项目 context；若不确定，问主人。
4. 如果有 working-os 自身改进建议，记录为系统复盘候选，不静默改 skill，除非主人明确要求执行。

## worklog 格式
```
# <日期> · Session · <简述>

## Topic
- 这次对话/任务围绕什么。

## Important Points
- 值得带走的重要点。

## Decisions
- 已形成的决定。

## To Route Into Context
- 应进入项目 context / skill / system review 的候选。

## Reflection Candidates
- 日复盘可提炼的认知、方法、坑、偏好。

## Follow-ups
- 后续要做的事。
```

## 原则
- `inbox/` 收未处理外部材料；`worklog/` 存 AI session 总结。
- worklog 可以保留中间思考的摘要，但不要把长对话逐字转存。
- 复盘时再决定哪些认知进入 `reviews/recurring.md` 或 `assets/`。
