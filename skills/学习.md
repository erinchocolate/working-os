# skill: 学习 —— 苏格拉底式探讨一个话题，产出学习笔记

**触发**：
- 「学习 <话题>」→ 进入探讨模式。
- 「学习 收尾」→ 把这轮对话提炼成学习笔记。

**目的**：和主人一起把一个技术/概念话题讲透，帮他真正理解；再把对话沉淀成可复用的学习笔记。

## 流程：探讨
1. 不要一上来灌大段定义。用**苏格拉底式**：先探主人已有的理解，从他的语境出发，一来一回地拆解。
2. 多用具体例子、类比；在关键处反问，确认他真的懂了再往下。
3. 主动指出常见误区、边界条件、和相邻概念的区别。

## 流程：收尾（产出笔记）
当主人说「学习 收尾」或一轮探讨自然结束时：
1. 把这轮对话提炼成**学习笔记**，存 `learning/<话题>/<话题>-notes.md`。
2. 笔记结构建议：一句话本质 → 关键点 → 例子/类比 → 易错点 → 还没搞懂的开放问题 → 延伸阅读。
3. 笔记是为「未来的主人」和「未来要用它的 AI」写的：精炼、可检索、能直接当背景喂给别的 skill。

## 与其他 skill 的关系
- 学习笔记可被「执行」「分享」按需引用作背景。
- 如果某次「分享」里你讲透了一段，主人可以手动把它晋升成这里的学习笔记。
