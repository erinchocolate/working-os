# skill: 复盘 —— 沉淀当天认知，跨情境复现晋升为认知资产（循环 B）

**触发**：
- 「复盘」→ 复盘今天。
- 「复盘 <日期>」→ 复盘指定日。
- 「复盘 照镜子」→ 跑照镜子流程。

**目的**：从每天的工作里提取认知；反复出现（且跨情境）的认知，提炼成可带走的「认知资产」；并反过来照见主人的长处与盲点。

## 先读
`assets/INDEX.md`（已有资产）、`reviews/recurring.md`（候选池现状）。

## 流程一：写当天复盘
1. 扫当天：和 AI 的对话、写的代码、做的分享、摄取的笔记。
2. 写 `reviews/daily/<日期>.md`。每条认知**结构化打标签**（不是纯散文，这是后续聚类的前提）：
   ```
   - type: 认知 / 坑 / 方法 / 偏好
     statement: 一句话认知
     evidence: 今天哪件事触发的
     context: 所属项目/情境
     tags: [关键词]
   ```

## 流程二：候选池增量聚类（不全量重扫历史）
把当天每条认知拿去和 `reviews/recurring.md` 里**现有聚类**做**语义匹配**（同一条认知每次措辞都不同，靠语义判断而非字符串）：
- **命中**已有聚类 → `count+1`、加 instance、更新 `last_seen`、若来自新项目则 `contexts` +1、顺手打磨 `canonical`。
- **未命中** → 开一条新候选（`count=1`）。

候选条目格式：
```
- canonical: 规范化的一句话认知（打磨到「可复用原则」高度）
  type: 认知 / 坑 / 方法 / 偏好
  count: N   first_seen: 日期   last_seen: 日期   contexts: [项目A, 项目C]
  instances: [每次原话 + 链回 reviews/daily/<日期>.md]
  status: 候选 / 已晋升 / 已搁置
```

## 流程三：晋升判定（平衡型 + 跨情境硬性要求，两扇门）
- **门一·复现型**：`≥3 个不同日子` **且** `跨 ≥2 个情境/项目` **且** 过非平凡门槛 → 标签 `已验证可迁移`。
  （**跨情境是硬门**：单项目内的反复只进该项目 context，**不进**资产库。）
- **门二·单次高信号型**：被主人（或你）显式标记「这条重要」的深刻顿悟 → 绕过复现/跨情境，过非平凡门槛即可 → 标签 `待验证`（可迁移性尚未被证明，诚实标注）。
- **非平凡门槛**：必须是关于「主人怎么思考/工作」的，不是一时状态或情绪（「今天累」不算）。

## 流程四：晋升（综合提炼 + 停下来问，不是复制原话）
够格的候选：
1. 把聚类**综合提炼**成一句脱离单次实例的资产。
2. 归类：心智模型 → `assets/mental-models.md`；通用坑 → `assets/pitfalls.md`；长处/盲点 → `assets/mirror.md`。
3. **停下来问主人**：接受 / 改写 / 否决 / 搁置。
4. 确认后写进 `assets/`，**链回证据**（哪几天的复盘支撑），候选标 `已晋升`。
5. 更新 `assets/INDEX.md` 精简索引。

## 衰减
长期未再复现的候选标 `已搁置`（休眠**不删**），保留主人曾经的关注点——可能正是已成长过去的阶段，照镜子会用到。

## 流程五：照镜子（「复盘 照镜子」或定期）
读 `assets/` + `reviews/` 全集 → 指出主人还没命名的长处/盲点 → 写入 `assets/mirror.md`，并和主人确认。
